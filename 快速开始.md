# 🚀 快速开始

## 立即发布新版本

```bash
./release.sh v1.0.0
```

就这么简单！脚本会自动：
- ✅ 更新版本号
- ✅ 提交到 Git
- ✅ 创建 Tag
- ✅ 推送到 GitHub
- ✅ 触发自动构建

## 构建平台

GitHub Actions 会自动构建：

| 平台 | 架构 | 产物 |
|------|------|------|
| Windows | x64 | `.msi`, `.exe` |
| macOS | ARM64 (M1/M2/M3/M4) | `.dmg` |
| macOS | x64 (Intel) | `.dmg` |
| Linux | x64 | `.AppImage`, `.deb` |

## 查看构建进度

1. 打开 GitHub 仓库
2. 点击 **Actions** 标签
3. 查看最新的工作流运行

## 下载安装包

构建完成后（约 15-25 分钟）：

1. 打开 GitHub 仓库
2. 点击 **Releases** 标签
3. 找到对应版本
4. 下载需要的安装包

## 常用命令

```bash
# 本地开发
pnpm tauri dev

# 本地构建（仅当前平台）
pnpm tauri build

# 发布新版本
./release.sh v1.0.0

# 查看 Git 状态
git status

# 查看所有 tags
git tag -l
```

## 首次使用前

1. **确保代码已推送到 GitHub**
   ```bash
   git remote -v  # 检查远程仓库
   ```

2. **（可选）配置签名密钥**
   - 运行 `python3 gen_key.py` 生成密钥
   - 在 GitHub 设置中添加 `TAURI_PRIVATE_KEY` 和 `TAURI_KEY_PASSWORD`
   - 更新 `src-tauri/tauri.conf.json` 中的 `pubkey`

3. **发布第一个版本**
   ```bash
   ./release.sh v1.0.0
   ```

## 版本号规范

遵循语义化版本（Semantic Versioning）：

- `v1.0.0` - 主版本.次版本.修订号
- `v1.0.1` - 修复 bug
- `v1.1.0` - 新增功能（向后兼容）
- `v2.0.0` - 重大更新（可能不兼容）

## 故障排除

### 构建失败？
1. 检查 Actions 日志中的错误信息
2. 确保所有依赖都正确配置
3. 验证版本号格式正确

### Tag 已存在？
```bash
# 删除本地 tag
git tag -d v1.0.0

# 删除远程 tag（谨慎！）
git push origin :refs/tags/v1.0.0
```

### 取消发布？
如果在推送前取消：
```bash
# 删除本地 tag
git tag -d v1.0.0

# 回退最后一次提交（如果需要）
git reset --soft HEAD~1
```
